Deploying to Vercel with Serverless Backend Functions

This project is now configured to deploy BOTH frontend and backend to Vercel as a single deployment.
The backend uses Vercel serverless functions (api/ folder) instead of a long-running server.

QUICK START

1. Push to GitHub
   git add .
   git commit -m "Convert backend to Vercel serverless functions"
   git push origin main

2. Deploy to Vercel
   - Go to https://vercel.com and sign in with GitHub
   - Click "Add New" > "Project"
   - Select your portfolio repository
   - Vercel will auto-detect the project structure
   - Configure:
     * Framework: Next.js (or Custom) - Select the root directory
     * Build Command: npm run build (for frontend)
     * Output Directory: frontend/dist
   - Click "Deploy"

3. Add Environment Variables in Vercel
   In your Vercel project settings, go to Settings > Environment Variables and add:
   
   CONTACT_TO=your-email@example.com
   SMTP_HOST=smtp.gmail.com
   SMTP_PORT=587
   SMTP_SECURE=false
   SMTP_USER=your-smtp-user@example.com
   SMTP_PASS=your-app-password-here

4. Verify Deployment
   - Frontend will be deployed to: https://your-project.vercel.app
   - API endpoints will be: https://your-project.vercel.app/api/contact
   - Test the contact form on the live site

PROJECT STRUCTURE FOR VERCEL

After the conversion:
- frontend/          - React Vite app (builds to frontend/dist)
- backend/
  - api/             - Serverless functions (becomes /api endpoints)
    - contact.js     - POST /api/contact endpoint
    - health.js      - GET /api/health endpoint
  - package.json     - Dependencies for serverless functions
  - .env             - Environment variables (NOT committed to GitHub)
  - .env.example     - Template for env vars

VERCEL.JSON CONFIG

The root vercel.json file tells Vercel:
- Where the backend code is (backend/ folder)
- How to build it (npm install)
- What to output (the api/ folder)

FRONTEND SETUP

The frontend uses /api/contact endpoint by default during production deployment.
Local development still uses http://localhost:3001 if you run the old Express server.

To use Vercel backend in frontend .env:
VITE_BACKEND_URL=

(Leave empty to use /api by default, or set to https://your-project.vercel.app for external calls)

API ENDPOINTS

/api/contact - POST
  Body: { name, email, message }
  Returns: { success, emailed, message }

/api/health - GET
  Returns: { ok, smtp }

TROUBLESHOOTING

If contact form doesn't work:
1. Check Vercel deployment logs (Deployments tab)
2. Verify environment variables are set in Vercel project settings
3. Check that SMTP_USER and SMTP_PASS are correct
4. Verify CONTACT_TO email is correct

If SMTP fails:
1. Use a Gmail App Password (not your regular password)
2. Enable 2-Step Verification on Gmail account
3. Generate App Password: https://myaccount.google.com/apppasswords
4. Add the 16-character password to Vercel as SMTP_PASS

NO LONGER NEEDED

- backend/server/ folder (old Express server - can be deleted)
- root backend package.json scripts for local dev (use api/ instead)
- PORT environment variable

GITHUB DEPLOYMENT FLOW

1. Make code changes locally
2. git push to GitHub
3. Vercel automatically builds and deploys
4. Frontend: new build in frontend/
5. Backend: new serverless functions in backend/api/
6. Everything live at your Vercel deployment URL

LOCAL TESTING (OPTIONAL)

If you want to test the serverless functions locally:
1. Install Vercel CLI: npm i -g vercel
2. Run: vercel dev
3. Access frontend at http://localhost:3000
4. API calls to /api/contact will work locally
